# 🛡️ Технический Паспорт Проекта (ctagg)

Этот документ является официальным руководством по дизайн-системе проекта, синхронизированным с макетами в Figma. Он предназначен для разработчиков и AI-ассистентов для поддержания визуальной и технической целостности интерфейса.

---

## Архитектура проекта (FSD)
Проект структурирован с элементами **Feature-Sliced Design (FSD)** для обеспечения модульности и масштабируемости:
- `src/components/layout/` — Глобальные структурные элементы (`Header`, `Footer`).
- `src/components/layout/header-modules/` — Изолированные микро-модули Хедера (Поиск, Телеметрия, Стрим, GameSwitcher).
- `src/components/ui/` — Переиспользуемые, "глупые" компоненты (`HubCard`, `GameCard`, `Carousel`, `Breadcrumbs`).
- `src/components/providers/` — Провайдеры контекста (`ThemeProvider`).

## 3. Игровые акценты

Система поддерживает два цветовых акцента, которые применяются к интерактивным элементам (рамки при наведении, иконки, ссылки).

- **EFT (По умолчанию):** Янтарный (`#E68E25`)
- **FRAGO (Тематический):** Бирюзовый (`#00CDAB`). Активируется глобально при добавлении класса `.theme-frago` к родительскому контейнеру.
- **ABI (Тематический):** Терракотовый (`#a34132`). Активируется глобально при добавлении класса `.theme-abi`.
- **GZW (Тематический):** Голубой (`#5FCAFF`). Активируется классом `.theme-gzw`.
- **ACTMAT (Тематический):** Оранжевый (`#FFA800`). Активируется классом `.theme-actmat` или `.theme-activematter`.
- **ARCR (Тематический):** Лавандовый (`#8A86FF`). Активируется классом `.theme-arcr` или `.theme-arcraiders`.
- **MRTHN (Тематический):** Лаймовый (`#C0FE04`). Активируется классом `.theme-mrthn` или `.theme-marathon`.
- **WDG (Тематический):** Золотой (`#E3BD74`). Активируется классом `.theme-wdg` или `.theme-wardogs`.
---

> 💡 **Важно про Tailwind CSS:** При стилизации интерактивных элементов, меняющих цвет в зависимости от темы игры (text, bg, border), используйте `[var(--primary)]` вместо стандартного `primary` (например, `text-[var(--primary)]`), чтобы избежать жесткой привязки к дефолтному янтарному HEX-коду конфигурации. Для прозрачности используйте функцию: `bg-[color-mix(in_srgb,var(--primary)_10%,transparent)]`.

## 5. Абсолютное позиционирование (внутри игровой карточки)

Элементы внутри большой игровой карточки (`348x564px`) позиционируются абсолютно для достижения пиксельной точности по макету Figma.

- **Логотип игры:**
  - **Координаты обертки:** `left-[46px] top-[440px] w-[256px] h-[96px]`.
  - **Архитектура:** Поддерживает три типа логотипов, задаваемых в `GAMES_DATA`:
    1.  **Многослойный (`multi-state`):** Два SVG/WebP изображения для состояний `default` и `hover` (используется в ARCR, Wardogs).
    2.  **CSS-маска (`mask`):** Одноцветный SVG-логотип, который динамически перекрашивается в акцентный цвет темы при наведении (EFT, FRAGO, ABI, GZW, ACTMAT, Marathon).
    3.  **Строка (string):** Простой путь к изображению в качестве fallback.
  - **Выравнивание:** Контент строго центрирован по осям с помощью Flexbox.
- **Подпись (название игры):**
  - **Координаты:** `top-[524px] left-0 w-full` (выравнивание по центру `text-center`).
  - **Стиль:** Адаптивный размер `text-[10px] xl:text-[12px]`, `font-blender-medium`, `uppercase`.
  - **Цвета:** По умолчанию `#9CA3AF`, при ховере — акцентный цвет темы `var(--primary)`, при INACTIVE — `#52525B`.

## 6. Интерактивные состояния и отказоустойчивость

### Эффекты наведения (Hover)
При наведении на активную карточку (`isInactive: false`) в течение **300мс** происходят следующие эффекты:
- **Кинематика:** Карточка плавно смещается вверх на `28px` (`transform: translateY(-28px)`).
- **Градиентная обводка:** Появляется `1px` обводка с вертикальным градиентом (от прозрачного вверху до `var(--primary)` внизу). Реализована через `background-clip: border-box` и CSS-маску (`mask-composite: exclude`) для идеального прилегания к скругленным углам.
- **Нижняя подсветка:** В нижней части карточки (в области `h-[28px]`) загорается радиальный градиент (`radial-gradient(ellipse_at_bottom, ...)`), создающий эффект неонового свечения от акцентного цвета `var(--primary)`.
- **Видео-фон:** Активируется многослойный видео-ховер с отказоустойчивостью.

### Отказоустойчивость видео (Graceful Degradation)
Для обеспечения плавной работы на медленных соединениях или при ошибках загрузки видео используется многослойный подход:
1.  **Базовый слой (Static):** Отображается статичное фоновое изображение `bg.webp`.
2.  **Слой наведения (Hover):** При наведении курсора поверх статического фона мгновенно появляется бэкап-изображение `bg-hover.webp`.
3.  **Видео-слой (Autoplay):** Параллельно с `bg-hover.webp` начинается загрузка и автовоспроизведение зацикленного видео `<video loop muted playsinline>`. Видео плавно появляется поверх `bg-hover.webp` после полной буферизации, создавая бесшовный эффект.

### Оптимизация медиа (YouTube и Иконки)
- **Иконки HubCard:** SVG-иконки в карточках хаба реализованы через `mask-image`. Это позволяет хранить один черно-белый файл в `/public/icons/`, который браузер динамически окрашивает в `var(--primary)` при наведении (hover).
- **Цветные иконки (Опционально):** Система автоматически распознает иконки, требующие сохранения оригинальных цветов (например, `.webp` аватарки торговцев, квесты `/02-quests/` или моды `/gun-modes/`), и рендерит их через стандартный тег `<img>` без наложения масок.
- **Ленивая загрузка YouTube:** Плееры на странице используют паттерн `LiteYouTube`. При загрузке страницы отображается только легковесное превью (`hqdefault.jpg`) с фальшивой кнопкой Play. Тяжелый `iframe` инжектируется в DOM только после явного клика пользователя.
- **Twitch API:** OAuth-токены для проверки статуса стрима кэшируются на сервере Next.js (`cachedToken`), что защищает приложение от блокировки за спам запросами (Rate Limiting).

### Каскадные анимации
Анимация `fade-in-up` используется повсеместно с инкрементальным `animation-delay` на основе `index` элемента, создавая эффект "водопада" при загрузке списков (карусель игр, карточки хаба).

### Глобальная командная панель (Command Palette)
В Хедер встроен компонент `TacticalSearch` (хоткей `CTRL+Q` / `CMD+Q`), который служит единой точкой входа. Он поддерживает **рекурсивную фильтрацию** по всем уровням навигационных ссылок (включая глубоко вложенные подменю) и асинхронный полнотекстовый поиск по базе данных предметов EFT (через GraphQL `tarkov.dev` API). Поиск усилен модулем **`search-engine.ts`**, который содержит обширный словарь игрового сленга (например, "ксюха", "мка", "ледекс", "биток"), позволяя находить предметы по их народным названиям с помощью продвинутой системы скоринга. Этот же API используется для динамического парсинга цен Барахолки и вычисления редкости лута в разделе "Нужные предметы" (`/eft/keepitems`).

## 7. Состояние «Неактивно» (INACTIVE)

Если для карточки установлен флаг `isInactive: true`, активируется режим "технических работ":

- **Фон:** Принудительно устанавливается фоновое изображение `bg-inactive.webp`.
- **Интерактивность:** Полностью отключаются все эффекты наведения (`:hover`), включая неоновые тени и смену рамок.
- **Видео:** Загрузка и воспроизведение видео-петель блокируются.
- **Навигация:** Ссылка (`<Link>`) блокируется, предотвращая переход на страницу неактивной игры.

## 9. Расширение системы

Проект спроектирован с учетом легкого добавления новых игровых разделов.

### Добавление новой игры

1.  **Создайте папку** в `/public/games/` с `id` новой игры (например, `/public/games/my-new-game/`).
4.  **(Опционально) Добавьте тему:** Если для игры нужен уникальный акцентный цвет, следуйте инструкции ниже.

### Добавление новой темы

1.  **Определите CSS-класс:** В файле `src/styles/globals.css` добавьте новый класс темы, например `.theme-my-new-game`.
2.  **Задайте акцентный цвет:** Внутри класса определите значение для переменной `--primary`:
    ```css
    .theme-my-new-game {
      --primary: #ваш-hex-код;
    }
    ```
3.  **Примените класс:** Укажите `themeClass: 'theme-my-new-game'` в объекте соответствующей игры в `GAMES_DATA`.

### Настройка Хедера (Header)
Чтобы глобальная шапка (Header) адаптировалась под новую игру:
1. Перейдите в `src/data/headerConfig.ts`.
2. Добавьте в `HEADER_DICTIONARY` новый объект по ID игры (например, `my-new-game`).
3. Укажите `searchPlaceholder` (текст поиска), `menuItems` (пункты меню) и `currencySymbol` (внутриигровую валюту). Хедер автоматически переключит логотип, дизайн и ссылки при переходе в этот раздел.
