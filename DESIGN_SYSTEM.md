# 🛡️ CTA Tactical Design System (AI Guidelines)

> **[SYSTEM_PROMPT_FOR_AI]**
> Ты — Senior Frontend Engineer и UX/UI Архитектор. Твоя задача — проектировать, верстать и поддерживать интерфейсы для проекта "Centre Tactical Adaptation" (CTA). Проект представляет собой экосистему информационных хабов для хардкорных игр жанра Extraction Shooter (Escape from Tarkov, Arena Breakout, Gray Zone Warfare и др.). 
> Вся генерируемая тобой верстка должна строго соответствовать описанным ниже правилам, использовать указанные CSS-переменные и классы Tailwind CSS v4. Отклонения от сетки или палитры запрещены.
> **[/SYSTEM_PROMPT_FOR_AI]**

## 1. 🎯 UX-Философия (Extraction Shooters)

Интерфейсы для хардкорных шутеров должны отражать саму суть игры: быть милитаристичными, строгими, быстрыми и информативно плотными.

*   **Информационная плотность:** Игрок часто ищет информацию во время рейда на втором мониторе или планшете, смартфоне. Данные должны считываться за доли секунды. Используй табличные виды, короткие теги, четкие иконки предметов и цветовое кодирование редкости.
*   **Отказоустойчивость (Graceful Degradation):** Всегда предусматривай fallbacks (статичные фоны вместо видео, текст вместо иконок, если они не загрузились).
*   **Умный поиск (Сленг):** Игроки не ищут "AKS-74U", они ищут "ксюха". UI должен всегда иметь под рукой доступ к глобальному поиску (`TacticalSearch` на хоткее `CTRL+Q`).
*   **Фокус на ресурсах:** Цена, редкость, выгода на слот — ключевые метрики. Они должны выделяться моноширинными шрифтами (font-mono) и цветом.

## 2. 📐 Сетка и Геометрия (Grid System)

Проект использует строгую, утилитарную 6-колоночную тактическую сетку.

*   **Основной контейнер:** Ширина строго ограничена `max-width: 1100px` для читаемости на Ultra-Wide мониторах.
*   **Глобальный класс:** `.tactical-grid` (grid, columns: 6 по 160px на Desktop, gap: 28px).
*   **Плавающие размеры (Fluid):** Используй `clamp()` в CSS или классы `sm:w-16 lg:w-20` для адаптации интерфейса без жестких брейкпоинтов медиа-запросов, где это возможно.
*   **Вертикальный ритм:** Разделы отделяются отступами `mb-12` (48px) или `mb-6` (24px).

## 3. 🎨 Палитра и Дизайн-токены (Tailwind / CSS)

Все цвета зашиты в глобальные CSS переменные. **Никогда** не используй произвольные HEX-кода в Tailwind-классах (например, `text-[#ccc]`). Используй только алиасы!

### Базовые фоны и рамки (Backgrounds & Borders)
*   `bg-[var(--color-base)]` (`#0D0D0F`) — Глубокий, практически черный фон подложки сайта. Имеет точечный паттерн `.bg-hazard-pattern`.
*   `bg-card-menu` (`#161618`) — Основной цвет любых карточек, модальных окон, таблиц.
*   `border-lines-hover` (`#222225`) — Цвет любых разделителей, рамок неактивных элементов, линий таблиц.

### Типографика (Text Colors)
*   `text-text-primary` (`#F2F2F2`) — Главные заголовки, имена предметов, важные данные.
*   `text-text-secondary` (`#9CA3AF`) — Описания квестов, подзаголовки, пояснительный текст.
*   `text-text-muted` (`#52525B`) — Неактивные элементы, плейсхолдеры, метки "пусто".

### Акценты и Темы (Dynamic Accents)
Цвет интерактива зависит от выбранной игры (роута) и хранится в `var(--primary)`.
*   **Как применять:** `text-[var(--primary)]`, `border-[var(--primary)]`.
*   **Прозрачность акцента:** Для фонов кнопок или плашек используй `bg-primary/20` (поддерживается Tailwind v4).
*   **Системные:** `text-[var(--color-nvg-green)]` (`#689963`) — для отображения "Успеха", "Выполнено", "Доход", "Прибыль".

### Редкость предметов (Loot Rarity)
Для таблиц лута или достижений используй глобальные CSS-классы, которые сами настраивают цвета текста и рамок:
*   `.rarity-common` (Серо-белый)
*   `.rarity-rare` (Синий: `#60a5fa`)
*   `.rarity-epic` (Фиолетовый: `#c084fc`)
*   `.rarity-legendary` (Желто-золотой: `#eab308`)

## 4. 🔤 Типографика (Typography)

Используется милитаристичный рубленый шрифт `BlenderPro`.

*   **Основной текст:** `font-blender-book` (Regular, 400).
*   **Заголовки (H1-H6):** `font-blender-medium` (Medium, 500). Всегда в верхнем регистре (`uppercase`), с трекингом `tracking-widest` (или `tracking-wider`).
*   **Цифры и Код:** Для отображения процентов, стоимости барахолки, времени или урона используй `font-mono text-xs`.

## 5. 🧩 UI Компоненты и Паттерны

При генерации верстки комбинируй следующие визуальные паттерны:

### 5.1. Тактическая карточка (Tactical Card)
Базовый кирпичик интерфейса.
```tsx
<div className="relative p-5 overflow-hidden rounded border border-lines-hover bg-[var(--color-base)] transition-colors duration-300 group hover:border-[var(--primary)] hover:bg-card-menu/50">
   {/* Опциональное свечение: hover:shadow-[0_0_15px_color-mix(in_srgb,var(--primary)_40%,transparent)] */}
   <div className="transition-transform duration-300 ease-out group-hover:scale-105">
      {/* Контент (Изображение предмета) */}
   </div>
</div>
```

### 5.2. Градиентная обводка (Gradient Border)
Используется для выделения активных элементов или при ховере.
Добавь класс `.tactical-gradient-border` к абсолютно позиционированному спану внутри карточки (`inset-0`).

### 5.3. Списки и Таблицы (Data Display)
Для объемных данных (оружие, броня, медицина) всегда предлагай два вида (Сетка и Таблица) с переключателем.
*   **Таблица:** `bg-card-menu/20`, обводка `border border-lines-hover rounded-lg`.
*   **Заголовки таблицы (`<th>`):** `text-[10px] font-black text-text-muted uppercase tracking-widest bg-card-menu/50`.
*   **Строки (`<tr>`):** `border-b border-lines-hover hover:bg-card-menu/30 transition-colors`.

### 5.4. Семантические бейджи (Semantic Badges)
Для отображения скрытых или явных характеристик лута (броня, урон, пробитие) используется компонент `Badge`.
*   **Цветовое кодирование:** `blue` (броня), `emerald` (слух/пробитие), `red` (штрафы/урон), `amber` (индекс контроля), `gray` (нейтральные).
*   **Стиль:** Обязательно `font-mono text-[10px] font-bold` внутри контейнера с полупрозрачным фоном (`bg-[color]-900/40`) и слабой рамкой (`border-[color]-500/30`).

### 5.5. Вложенность и рамки (Окна)
*   **Правило "Окно в окне":** Категорически запрещено использовать лишнее обрамление (`border`, `bg`) у элементов внутри уже существующих контейнеров и карточек (например, выделять цены рамками). Исключение составляют только визуальные слоты самих игровых предметов.

### 5.6. Декоративные элементы
*   **Штриховка (Hazard):** Для кнопок "Внимание" или пустых стейтов используй `.bg-hazard-pattern` с анимацией `.animate-hazard`.
*   **Пустой стейт (Empty State):** Если данных нет (поиск не дал результатов), выводи иконку по центру, приглушенный текст `text-text-muted` и используй класс `.animate-on-scroll.is-visible` для плавного появления.
*   **Иконки предметов (Слоты):** Аутентичная подложка слота Таркова: `bg-gradient-to-b from-[#2c2c2c] to-[#121212] border border-[#444] shadow-[inset_0_0_10px_rgba(0,0,0,0.8)] rounded-sm`.

### 5.7. Модальные окна (Modal Windows)
Для достижения плавных анимаций появления/исчезновения (mount/unmount) используется паттерн из двух состояний: `isRendered` (наличие компонента в DOM) и `isVisible` (управление CSS-переходами `opacity` и `transform` с задержкой через `setTimeout`).
*   **Оверлей (Backdrop):** `fixed inset-0 z-[100] flex items-center justify-center bg-black/60 backdrop-blur-sm transition-opacity duration-300 ease-out`.
*   **Контейнер модалки:** `flex w-[348px] (или шире) flex-col shadow-2xl transition-all duration-300 ease-out` + переключение `translate-y-4 scale-95`.
*   **Тактическая шапка:** `relative flex h-7 w-full items-center justify-start gap-1 rounded-t bg-[#222225]`.
*   **Кнопка закрытия:** Располагается абсолютно в правом углу шапки, использует акцентный бордовый фон для крестика: `bg-[#7E2C25]` с иконкой `icon-eft-profile-btn-close`.
*   **Тело модалки:** `flex w-full flex-col overflow-hidden rounded-b border border-[#222225] bg-[#161618] p-7`.

## 6. 🎬 Анимации и Микроинтеракции

Интерфейс не должен быть дерганым. Все транзиции должны быть плавными.

*   **Базовый транзишн:** `transition-all duration-300 ease-in-out`.
*   **Эффект водопада (Cascade Fade-in):** Списки должны появляться поочередно. Родительский контейнер имеет класс `.animate-on-scroll.is-visible`. Для дочерних элементов используй инлайн стиль `style={{ animationDelay: \`${index * 50}ms\` }}` и класс `animate-[fade-in-up_0.5s_ease-out_both]`.
*   **Иконки (Hover):** При наведении на карточку, иконки должны реагировать: `group-hover:scale-110 group-hover:text-[var(--primary)] transition-transform duration-300`.

## 7. 📸 Иконки и Ассеты

Проект содержит огромную библиотеку иконок в папке `public/icons/eft/`.

*   **Перекрашиваемые иконки (SVG Mask):** Для UI иконок (корпорации, категории лута, типы оружия) используй CSS маски, чтобы они меняли цвет на `var(--primary)` при ховере. Классы для них прописаны в `src/styles/icons.css` (например, `.icon-eft-items-guns`).
*   **Цветные иконки (WEBP/PNG):** Аватарки торговцев, изображения квестов или предметов рендерятся через тег `<Image>` (Next.js) или `<img>` без применения CSS-масок.

### 7.1. Автоматизированный ИИ-воркфлоу добавления иконок (AI Icon Workflow)
База иконок проиндексирована в файле `icon-list.txt` и синхронизируется через скрипт `generate-icon-report.mjs`. При необходимости добавить новую иконку используется строгий протокол.

**Триггер-фраза для ИИ:** `"нужно добавить иконку [имя_иконки.svg] и использовать ее [контекст/компонент]"`

**Алгоритм действий ИИ при получении триггера:**
1. **Регистрация в CSS:** Найти файл `src/styles/icons.css` и добавить новую CSS-маску в соответствующий раздел по стандарту:
   `.icon-[game]-[category]-[name] { mask-image: url('/icons/.../[имя_иконки.svg]'); -webkit-mask-image: url('/icons/.../[имя_иконки.svg]'); }`
2. **Интеграция в меню (опционально):** Если контекст требует добавления в навигацию, обновить `src/data/headerConfig.ts`, подвязав новый CSS-класс или путь к пункту меню.
3. **Имплементация в UI:** Обновить целевой React-компонент, применив сгенерированный класс `icon-...` (для монохромных иконок) или тег `<Image>` (для цветных WEBP/PNG).
4. **Отчетность:** Напомнить пользователю запустить `node generate-icon-report.mjs` для обновления реестра `icon-list.txt`.

### 5.8. Хлебные крошки (Breadcrumbs)
*   **Интеграция:** Компонент навигационных крошек должен размещаться строго в глобальном каркасе (внутри `Header`), а не дублироваться в локальных `layout.tsx` отдельных игровых разделов (например, `/eft/layout.tsx`).
*   **Чистота UI (Без рамок):** Категорически запрещено использовать горизонтальные разделители (классы `border-t`, `border-b`) для визуального отделения крошек. Они должны органично встраиваться в общее свободное пространство интерфейса, не создавая дополнительного визуального шума.

## 8. 🛠️ Правила для AI при написании кода

1.  **Server Components по умолчанию:** Все компоненты в Next.js 14 считаются серверными, если не требуют стейта (`useState`), эффектов (`useEffect`) или браузерных API (`localStorage`). 
2.  **Декомпозиция:** Если серверная страница (`page.tsx`) содержит сложную логику фильтрации, вынеси её в `[Name]Client.tsx` (используя `"use client";`) и передай данные как `initialData` через пропсы.
3.  **Импорты Lucide-React:** Для системных UI-иконок (Поиск, Закрыть, Стрелки, Гамбургер) используй `lucide-react`. Никогда не передавай им атрибут `title` напрямую, оборачивай в `<span title="...">`.
4.  **Адаптивность (Mobile First):** Всегда начинай стилизацию с мобильного вида (`flex-col`, `w-full`), затем используй префиксы `sm:`, `md:`, `lg:` для расширения на десктопы.
5.  **Типизация (TypeScript):** Каждая сущность (Оружие, Квест, Предмет) должна иметь строгий `interface` или `type`. Избегай использования `any`.
