# 📜 История изменений проекта (Changelog)

Все заметные изменения, внесенные в проект `ctagg`, будут документироваться в этом файле.
Формат основан на Keep a Changelog.

## [3.4.0] - Детальные страницы предметов и Торговля

### 🚀 Добавлено (Added)
- **Детальная страница предмета (`ItemDetailsPage`):** Реализована модульная структура отображения детальной информации о предмете.
- **Динамические модули характеристик:** Созданы `WeaponModule`, `ArmorModule`, `MedicalModule`, `ContainerModule` (с рендерингом внутренней сетки) для семантического вывода специфичных свойств лута.
- **Торговый модуль (`TraderModule`):** Разработана система вывода предложений от торговцев и барахолки (`buyFor`, `sellFor`), с автоматическим определением валюты ($ / € / ₽) и сортировкой по выгоде.
- **Утилиты форматирования (`formatters.ts`):** Внедрены `formatCompactNumber` и `getCurrencySymbol` для "тактического" вывода цен (например, `1,5К ₽` или `100 $`), обернутые в `title` для просмотра точной стоимости.
- **Продвинутая фильтрация:** Добавлен фильтр "Доступно мне" в каталог, считывающий состояние глобального профиля (Zustand) для скрытия недоступного лута по уровню лояльности торговцев или уровню ЧВК (< 15 для барахолки).

## [3.3.0] - UI/UX Каталога Предметов

### 🚀 Добавлено (Added)
- **Экономика в плитках:** В карточки предметов (режим "Сетка") добавлен вывод цены за один слот для быстрого определения выгоды.
- **Динамическая редкость:** Реализован маппинг цветовых подложек карточек (`backgroundColor`) на основе API `tarkov.dev`. Фоновое свечение теперь отображается в каталоге (`ItemTile`) и глобальном поиске (`SearchItemCard`).
- **Скелетная загрузка (Skeleton):** Добавлена плавная анимация состояния ожидания данных для табличного и плиточного видов в `ItemsCategoryClient`, предотвращающая скачки интерфейса.

### ♻️ Изменено (Changed)
- **Оптимизация таблиц:** Внедрены адаптивные ограничения ширины для ячеек с названиями предметов. Длинные имена теперь корректно обрезаются многоточием (`truncate`), сохраняя целостность верстки.
- **Рефакторинг (UI):** Массивный код рендеринга карточки предмета вынесен из `ItemsCategoryClient` в изолированный и переиспользуемый компонент `ItemTile.tsx`.
- **Бейджи брони:** Удален несуществующий 7-й класс брони. Иконки классов брони (1-6) значительно увеличены в размерах (до 22x22px) и получили цветовую идентификацию редкости на базе новой функции `getArmorClassColor`.

### 🐛 Исправлено (Fixed)
- Исправлена проблема растягивания таблиц на мобильных устройствах из-за слишком длинных названий модификаторов и оружия.

## [3.2.0] - Интерактивный каталог предметов

### 🚀 Добавлено (Added)

- **Семантические бейджи:** Разработан переиспользуемый UI-компонент `Badge.tsx` для вывода динамических метрик (урон, пробитие, класс брони, скрытый индекс контроля).

- **Анимации карточек:** Логика увеличения карточек (`hover:scale`) перенесена на внутренний контейнер для предотвращения перекрытия соседних элементов в сетке.

## [3.1.0] - Интерактивные карты и Достижения

### 🐛 Исправлено (Fixed)
