## 🏗️ 1. ТЕХНОЛОГИЧЕСКИЙ СТЕК И АРХИТЕКТУРА (CORE_ARCHITECTURE)

- **Фреймворк:** Next.js 14+ (App Router). По умолчанию все компоненты — Server Components. Директива `"use client"` применяется ТОЛЬКО для интерактивных узлов (хуки стейта, обработчики событий, браузерные API).
- **Стилизация:** Tailwind CSS v4. Строгий порядок классов (Позиционирование -> Размеры -> Типографика -> Цвета -> Адаптивность).
- **Глобальный стейт:** Zustand (профили, настройки, UI-переключатели).
- **Типизация:** TypeScript Strict. Использование `any` запрещено. Применяются Discriminated Unions для сложных объектов (например, `tarkov-items.ts`).
- **Структура (FSD-lite):** 
  - `src/components/ui/` — Глупые переиспользуемые атомы (кнопки, бейджи).
  - `src/components/features/` — Умные компоненты с бизнес-логикой.
  - `src/components/layout/` — Каркасы (Header, Footer, модули навигации).
  - `src/lib/` и `src/data/` — Словари, API-интеграции, утилиты.
- **Оптимизация (Strict Rules):** 
  - Никаких спиннеров загрузки — только Skeleton Screens (`animate-pulse`).
  - Для длинных списков (таблицы предметов, квесты) — использование виртуализации.
  - Все изображения — через `next/image` или кастомный Cloudflare Loader (если указано). Иконки-маски через CSS `mask-image`.

## 🎨 2. ДИЗАЙН-СИСТЕМА "NIGHTFALL" (UI/UX)

**[UI_RULE]: Категорический запрет на произвольные HEX-коды в Tailwind (например, `bg-[#121212]`). Использовать исключительно токены!**

### 2.1 Токены Цветов (CSS Variables)
- **Фоны:** 
  - `bg-[var(--color-base)]`: Глубокий базовый фон холста.
  - `bg-[var(--color-card-menu)]`: Карточки, модальные окна, панели (Elevation 1).
- **Рамки и Разделители:** 
  - `border-[var(--color-lines-hover)]`: Неактивные рамки, линии таблиц.
- **Текст:**
  - `text-[var(--color-text-primary)]`: Заголовки, главные данные.
  - `text-[var(--color-text-secondary)]`: Описания, метаданные.
  - `text-[var(--color-text-muted)]`: Плейсхолдеры, микро-текст, пустые стейты.
- **Акценты (Динамические по игре):**
  - `text-[var(--primary)]`, `border-[var(--primary)]` — динамический акцент (Янтарный для EFT).
  - Прозрачный акцент: `bg-[color-mix(in_srgb,var(--primary)_20%,transparent)]`.
- **Семантические цвета (Опционально):**
  - `text-[var(--color-nvg-green)]` — Успех / Прибыль.
  - `text-red-500` / `bg-red-900/40` — Ошибка / Урон / Провал.

### 2.2 Типографика и Сетка
- **Шрифты:** `font-blender-book` (основной), `font-blender-medium` (для `uppercase` заголовков `tracking-widest`), `font-mono text-xs` (числа, цены, статы — обязательно!).
- **Сетка (Tactical Grid):** Базовый шаг 4px (`p-1`). Основной контейнер `max-w-[1100px]`, `.tactical-grid` (6 колонок по 160px, gap 28px).

### 2.3 Паттерны и Микроинтеракции
- **Анимации:** Водопадное появление `.animate-on-scroll.is-visible` с задержками. Базовый переход: `transition-all duration-300 ease-out`.
- **Ховер карточек:** Подъем содержимого (`group-hover:scale-105`), появление градиентной рамки (`.tactical-gradient-border`) или свечения.
- **Пустые состояния (Empty States):** Обязательны. Иконка по центру, текст `text-text-muted`, кнопка призыва к действию.

## 🤖 4. ПРАВИЛА ГЕНЕРАЦИИ КОДА ДЛЯ ИИ (STRICT PROTOCOLS)

6. **Синхронизация Иконок:** При запросе `"нужно добавить иконку [X]"` ИИ обязан сначала сгенерировать CSS-класс в `src/styles/icons.css`, пробросить его в `headerConfig.ts` (если это пункт меню) и только затем применять в компоненте.
